# llm-code (Transformers + AWQ)

## Почему медленнее vLLM

vLLM даёт непрерывный батчинг, PagedAttention и узкоспециализированные CUDA-ядра. Здесь — `transformers.generate()` + gptqmodel (часто `torch_awq`), это **на порядок медленнее** на том же GPU. Для максимальной скорости снова поднимайте **vLLM**, если образ/драйвер позволяют.

## Что ускорить без смены стека

| Переменная | Смысл |
|------------|--------|
| `LLM_CODE_AWQ_BACKEND=marlin` | Часто быстрее `torch_awq`; на новых GPU может не загрузиться — тогда `torch_awq`. |
| `LLM_CODE_ATTN_IMPLEMENTATION=sdpa` | По умолчанию; при ошибках загрузки/инференса попробуйте `eager`. |
| `MAX_NEW_TOKENS` | Меньше токенов → быстрее ответ (например `512`). |
| `LLM_CODE_QWEN_THINKING=0` | По умолчанию: **выкл.** «мышление» Qwen3 (иначе ответы намного длиннее и медленнее). `=1` — включить. |

Также при старте включаются TF32 для matmul на CUDA.
